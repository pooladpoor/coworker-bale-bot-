# 🤖 Bale AI Answer Bot

ربات بله برای پاسخ‌دهی خودکار به سوالات تصویری با استفاده از چند مدل هوش مصنوعی به‌صورت هم‌زمان (Gemini، Claude، GPT)، با معماری کاملاً async و پشتیبانی از چت خصوصی و گروه. هر مدل کاملاً مستقل تنظیم می‌شود و به هر پلتفرم سازگار با OpenAI API (CometAPI، OpenRouter، یا هر gateway دیگری) قابل اتصال است — وابسته به هیچ پلتفرم خاصی نیست.

## ✨ امکانات

- ارسال یک عکس سوال و دریافت پاسخ تشریحی از **سه مدل هوش مصنوعی به‌صورت موازی** (Gemini، Claude، ChatGPT)
- خروجی یک **گزارش HTML** سه‌ستونه با رندر کامل فرمول (MathJax) و مارک‌داون، راست‌به‌چپ
- پردازش کاملاً **ناهمزمان (async)** با صف کار و pool ثابتی از worker‌ها → پاسخ‌گویی هم‌زمان به چند کاربر بدون معطلی
- Retry خودکار با backoff نمایی برای خطاهای شبکه‌ای
- قابل استفاده هم در **چت خصوصی** و هم در **گروه** (در گروه فقط با تگ کردن ربات فعال می‌شود)
- **کنترل دسترسی اختیاری**: می‌توانید فقط چند آیدی عددی خاص را مجاز به استفاده از ربات کنید؛ بقیه پیام رد دریافت می‌کنند
- تمام تنظیمات (تعداد worker، timeout‌ها، مدل‌ها، رفتار گروه و ...) از طریق فایل `.env`
- هر مدل با **base URL و کلید API مستقل** فراخوانی می‌شود؛ می‌توانید هر مدل را روی یک پلتفرم متفاوت اجرا کنید و مصرف هرکدام را جدا رصد کنید

## 🧱 معماری

```
polling loop  --enqueue-->  صف کار (asyncio.Queue)  --مصرف توسط-->  N worker موازی
```

حلقه‌ی polling فقط پیام‌های جدید را می‌گیرد و در صف می‌گذارد؛ پردازش واقعی (دانلود عکس، فراخوانی هم‌زمان دو مدل، ساخت HTML، آپلود) توسط چند worker مستقل انجام می‌شود، بنابراین کاربران هم‌زمان بدون مسدود کردن یکدیگر پاسخ می‌گیرند.

## 🚀 راه‌اندازی

```bash
git clone <repo-url>
cd <repo-folder>
python3.12 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

cp .env.example .env
# BALE_TOKEN و برای هر مدل سه مقدار (BASE_URL / API_KEY / MODEL) را در .env تنظیم کنید

python main.py
```

## ⚙️ ساختار پروژه

| فایل | مسئولیت |
|---|---|
| `main.py` | نقطه‌ی ورود؛ حلقه‌ی polling و مدیریت worker‌ها |
| `config.py` | تنظیمات پروژه (از `.env`) |
| `bale_client.py` | ارتباط async با Bale Bot API |
| `ai_client.py` | ارتباط async با سه مدل هوش مصنوعی، هرکدام روی پلتفرم/کلید مستقل خودش |
| `html_renderer.py` | ساخت گزارش HTML سه‌ستونه (مارک‌داون + MathJax) |
| `worker.py` | خط پردازش هر سوال از ابتدا تا انتها |
| `queue_manager.py` | صف کار |
| `utils.py` | ابزارهای مشترک (لاگ، retry، تشخیص تگ) |
